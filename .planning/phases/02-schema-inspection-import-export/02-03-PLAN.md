Create internal/output/ddl.go with GetCreateTable function:

1. Function signature: GetCreateTable(db *gorm.DB, tableName string) (string, error)

2. Logic:
   - Call database.GetTableColumns to get column information
   - Parse column info to build CREATE TABLE statement:
     - Map MySQL types to generic SQL types (VARCHAR, INT, DATETIME, etc.)
     - Handle NULL/NOT NULL based on IS_NULLABLE
     - Handle DEFAULT values
     - Handle AUTO_INCREMENT (check Extra field for "auto_increment")

   - Generate: CREATE TABLE `name` ( col1 TYPE [NOT NULL] [DEFAULT x], ... ) ENGINE=...;

```sql

  `id` int NOT NULL AUTO_INCREMENT,
  `name` varchar(255) NOT NULL,
  `email` varchar(255) DEFAULT NULL,
  `created_at` datetime DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

4. For MVP, focus on basic column definitions. Complex constraints can be added later.

<name>Task 3: Implement export command</name>
<files>cmd/export.go</files>
<action>
Create cmd/export.go with exportCmd implementing:

1. Flags:
   - --query (string): SQL query to execute and export
   - --table (string): Table name to export (structure + data)
   - --output (string): Output file path (required)
   - --format (string): Output format - "insert" (default) or "ddl"

2. Command logic in runExport function:
   - Validate: either --query or --table must be provided, not both
   - Validate --output is provided
   - Validate --format is "insert" or "ddl"
   - Open database connection
   - Route based on flags:

   a) --query provided:
      - Execute query with db.Raw(query)
      - Get rows
      - If format=="insert": call output.ToInsert(rows, extractedTableName)
      - Write result to --output file

   b) --table provided:
      - If format=="ddl": call output.GetCreateTable(db, tableName), write to file
      - If format=="insert": SELECT * FROM table, then call output.ToInsert

3. File writing:
   - Use os.WriteFile with 0644 permissions
   - Optionally add header comment: -- Exported by db-cli on <timestamp>

4. Error handling:
   - Same pattern as exec.go
   - Clear error messages for file write failures
</action>
