## CI/CD Pipeline

- [ ] **CI/CD-01**: Tag 推送后自动触发多架构构建
  - 触发条件：push tags `v*`
  - 构建平台：Windows/macOS/Linux
  - 构建架构：amd64/arm64
  - CGO 禁用：纯 Go 二进制

- [ ] **CI/CD-02**: 自动上传构建产物到 GitHub Releases
  - 每个平台的二进制作为 release asset 上传
  - 命名格式：`db-cli-{os}-{arch}[.exe]`
  - Release 描述自动生成 changelog

- [ ] **CI/CD-03**: 自动更新 db-skill 仓库的二进制文件
  - 使用 PAT 跨仓库推送
  - 更新 `binaries/` 目录下的文件
  - 创建版本化目录 `binaries/vX.Y.Z/`
  - 更新 latest 副本和 VERSION 文件

---

## Security & Configuration

- [ ] **SEC-01**: 配置跨仓库访问 PAT
  - 创建 Fine-grained PAT 或 Classic PAT
  - 限制到 db-skill 仓库
  - 添加到 db-cli 仓库 Secrets: `CROSS_REPO_PAT`

- [ ] **SEC-02**: 最小权限原则
  - PAT 只需要 `Contents: Read & Write` 权限
  - 不需要 workflow 修改权限
  - 定期轮换 PAT

## Out of Scope (v1.1)

- **交互式 REPL 模式** — 保持纯子命令设计
- **环境配置管理** — 每次执行时手动指定连接参数
- **密码存储** — 不保存敏感信息到配置文件
- **轮询同步** — 使用实时推送而非轮询下载
